# Agent 浏览器与工作流：检索结果需要可追溯

> 本文为 AnyViking Research 项目 demo 使用的合成资料，不代表真实新闻报道。

## 摘要

Agent 浏览器和自动化工作流正在把搜索、阅读、整理和执行连接起来。用户希望 AI 不只是回答问题，还能打开资料、比对信息、写出草稿并保存中间结果。

## 主要观察

早期 Agent 产品常见问题是过程不透明。用户看到最终答案，却不知道答案来自哪些页面、哪些文件，甚至不知道 AI 是否跳过了关键资料。对调研、法务、投研和课程学习这类场景来说，这种不透明会降低信任。

更稳妥的工作流应该保存每一步证据：检索问题、检索范围、返回 URI、摘要、分数和后续人工筛选结论。`viking://` 这类资源路径对 Agent 很有帮助，因为它让本地资料拥有可引用地址。

Agent 浏览器适合处理公开网页和实时资料，本地检索工具适合处理已经收集好的项目资料。两者结合后，可以形成一条闭环：公网发现资料，人工或脚本保存为 markdown，本地工具建立索引，Agent 后续围绕本地资料继续问答和写报告。

## 对开发者的启发

在开发检索 Skill 时，第一版不必急着做复杂的自主规划。更重要的是让输入和输出稳定：输入有明确 scope，输出有 JSON 和 URI，报告草稿能保留检索痕迹。
