# 开源检索工具生态：公网搜索、本地语料与 Skill 包装

> 本文为 AnyViking Research 项目 demo 使用的合成资料，不代表真实新闻报道。

## 摘要

开源检索工具正在形成三类互补方向：公网搜索连接器、本地语料数据库和面向 Agent 的 Skill 包装。三者解决的问题不同，组合起来才更接近完整调研系统。

## 主要观察

公网搜索连接器负责发现新资料。它们面向网页、新闻、论坛和论文，优势是时效性强，缺点是结果容易受网络质量和页面结构影响。

本地语料数据库负责沉淀资料。用户可以把已经确认有价值的资料保存为 markdown 或其他文档格式，导入本地索引。OpenViking 这类工具的价值在于资源管理和语义检索，而不是替代公网搜索。

Skill 包装层负责让 AI Agent 知道如何调用工具。一个好的 Skill 应该说明何时使用、如何传参、返回什么格式、哪些文件不能提交，以及如何处理错误。

## AnySearch 与 OpenViking 的分工

AnySearch 这类工具更像公网搜索入口，适合“找资料”。OpenViking 更像本地语料后端，适合“检索已保存资料”。AnyViking Research 可以把后者包装成稳定命令，未来再接入 AnySearch 作为上游资料来源。
